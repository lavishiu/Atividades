# Atividade 02 — Site do Grêmio (cards com flex-wrap + elemento fixo)

## Objetivo

Dar continuidade ao website do Grêmio utilizando a base fornecida: criar novos cards de notícias mantendo o padrão e organizar os cards em linhas com Flexbox (`flex-wrap: wrap`), incluindo um elemento fixo no canto inferior direito da tela.

## Capacidades avaliadas

- CT7: Aplicar CSS (Flexbox e posicionamento fixo) com consistência.
- CS3: Organização do projeto e registro de etapas no GitHub.

## Entregas

- [ ] `index.html` (base + novos cards)
- [ ] `style.css` (flex-wrap e estilos do elemento fixo)
- [ ] `assets/` com imagens (se usar)
- [ ] Pelo menos **5** cards `.noticia` (2 da base + 3 criados)
- [ ] Um elemento fixo no canto inferior direito da tela

## Critérios técnicos mínimos (checklist)

- [ ] Os 3 novos cards repetem o padrão do card existente (`div.noticia` + `img.img_noticia` + `h3` + `p`).
- [ ] A seção `.noticias` usa `display: flex` e `flex-wrap: wrap`.
- [ ] Os cards ficam centralizados quando quebram linha (`justify-content: center`).
- [ ] O elemento fixo usa `position: fixed` e fica no canto inferior direito (`right` e `bottom`).
- [ ] CSS em arquivo separado e código organizado.

## Rubricas

### (CT7)

- Insuficiente: Cards fora do padrão; flex-wrap não funciona (não quebra linha ou fica desorganizado); elemento fixo ausente ou não permanece fixo; uso confuso de classes/seletores.
- Básico: Cards criados com inconsistências; flex-wrap parcial (quebra, mas alinhamento/centralização irregular); elemento fixo existe, mas posição/comportamento incorretos.
- Adequado: Pelo menos 5 cards com padrão coerente; flex-wrap e centralização funcionam; elemento fixo permanece no canto inferior direito ao rolar; CSS organizado e reaproveitável.
- Avançado: Além do adequado, mantém consistência visual (espaçamentos/alinhamentos); evita repetição e “remendos”; demonstra refinamento sem perder o padrão da base.

### (CS3)

- Insuficiente: Projeto desorganizado; faltam entregas; commits inexistentes ou commit único.
- Básico: Organização parcial; commits poucos/genéricos; pequenas falhas de entrega/visualização.
- Adequado: Estrutura clara; commits por etapa; entrega completa e fácil de validar.
- Avançado: Além do adequado, evidencia planejamento e revisão final; commits mostram claramente a evolução (etapa → commit → resultado).
